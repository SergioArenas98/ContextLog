- `feature_list_screen.dart`
- `feature_detail_screen.dart`
- `feature_form_screen.dart`
- `feature_summary_tab.dart`
- `widgets/feature_card.dart`

## Naming conventions

| Pattern | Example |
|---|---|
| Feature module | `lib/features/find/` |
| Table class | `FindsTable` |
| Repository class | `FindRepository` |
| Model class | `FindModel` |
| Validator class | `FindValidator` |
| Provider file | `find_providers.dart` |
| List provider | `findsByFeatureProvider` |
| Repository provider | `findRepositoryProvider` |
| Validator provider | `findValidatorProvider` |
| Screen/tab widget | `FindListTab`, `FindFormSheet` |

## Important notes

- `*.freezed.dart` and `app_database.g.dart` are **generated files** вЂ” do not edit manually; regenerate with `dart run build_runner build --delete-conflicting-outputs`
- All feature modules follow the same three-layer structure; deviating will break the pattern
- `core/widgets/` contains only shared, reusable widgets; feature-specific widgets stay in their module
- The `harris_matrix` feature's `presentation/widgets/` contains the matrix painter and the relation form вЂ” there is no separate `harris_matrix_list_tab` because the matrix is a canvas, not a list
